# 钉钉机器人消息接收问题排查指南

## 🎯 问题现象
- ✅ 代码配置正常（Client ID、Secret 都有）
- ✅ 启动成功，显示"已连接"
- ❌ 发送消息后无任何响应
- ❌ 日志中看不到 `[Stream] ⭐ 收到机器人消息`

**两个版本都一样** → 说明问题在**钉钉开放平台配置**，不是代码问题。

---

## 🔍 完整排查清单

### ✅ 第一步：确认机器人状态

#### 1. 登录钉钉开放平台
https://open.dingtalk.com/

#### 2. 进入你的应用
- 找到"应用开发" → "企业内部开发"
- 点击你的机器人应用

#### 3. 检查关键配置

**A. 应用状态**
```
❌ 开发版 → 不稳定，可能收不到消息
✅ 正式版 → 稳定，必须发布
```

**操作：**
- 如果是开发版，点击"发布"按钮
- 填写发布信息
- 提交并等待审核（通常几分钟）

**B. 消息接收地址**
```
Stream 模式：不需要填写回调地址
但需要确保已启用 Stream 模式
```

---

### ✅ 第二步：检查机器人权限

#### 1. 进入"权限管理"页面

**必需权限：**
```
✅ 消息接收
✅ 通讯录只读权限（获取发送者信息）
✅ 个人信息获取权限
```

**操作：**
- 勾选这些权限
- 点击"申请权限"
- 等待审批通过

---

### ✅ 第三步：配置消息接收

#### 1. 进入"机器人"页面

**关键配置：**

**A. 消息接收**
```
类型：Stream 模式（不用填回调 URL）
Topic：勾选"机器人消息"
```

**B. 消息接收范围**
```
选项 1：全部员工（推荐测试）
选项 2：指定员工（稳定后使用）
```

**重要：** 确保你的账号在接收范围内！

---

### ✅ 第四步：添加机器人到会话

#### 方式 1：单聊（推荐先测试）

1. 在钉钉中搜索你的机器人
2. 发送消息
3. 如果第一次需要授权，点击同意

#### 方式 2：群聊

1. 创建或进入一个群
2. 添加机器人到群
3. @机器人 发送消息

**注意：** 群聊时需要 @ 机器人才能触发

---

### ✅ 第五步：验证配置

#### 运行诊断工具
```bash
npm run check:dingtalk
```

应该显示：
```
✅ Client ID 格式正常
✅ Client Secret 格式正常
✅ 未发现明显配置问题
```

#### 重启服务
```bash
npm run dingtalk:multi
```

观察启动日志：
```
[Stream] ✅ 连接已建立
[Stream] ✅ 已连接到钉钉服务器
[心跳] 12:00:00 - 连接正常
```

#### 发送测试消息

在钉钉中发送：
```
测试消息 123
```

**预期日志：**
```
[Stream] ⭐ 收到机器人消息
[Stream] 完整数据: {...}
[解析] senderId=xxx, content=测试消息 123
```

**如果看到 → 说明配置成功！**
**如果看不到 → 继续下一步**

## 🐛 常见问题和解决方案

### 问题 1：机器人是开发版

**症状：** 偶尔能收到消息，大部分时候收不到

**解决：**
1. 进入开放平台
2. 点击"发布应用"
3. 等待审核通过
4. 重新启动服务

---

### 问题 2：机器人未添加到会话

**症状：** 连接正常，但完全收不到消息

**解决：**
1. 在钉钉中搜索机器人名称
2. 点击添加为好友
3. 或者在群聊中添加机器人
4. 确保添加成功

---

### 问题 3：权限不足

**症状：** 日志显示认证失败或权限错误

**解决：**
1. 检查"权限管理"页面
2. 确保有"消息接收"权限
3. 重新申请权限
4. 重启服务

---

### 问题 4：消息接收范围限制

**症状：** 部分人能收到，部分人收不到

**解决：**
1. 进入"机器人"配置
2. 找到"消息接收范围"
3. 改为"全部员工"（测试时）
4. 保存并重启

---

### 问题 5：Stream 模式未启用

**症状：** 连接成功但收不到消息

**解决：**
1. 进入"机器人"配置
2. 确保"消息接收"类型是 Stream
3. 不要填写 HTTP 回调地址
4. 保存配置

---

```bash
curl -I https://api.dingtalk.com

# 应该返回 HTTP/1.1 200 OK
```

### 检查防火墙

```bash
# Windows
- 检查防火墙是否阻止 Node.js
- 检查公司网络是否有代理

# 解决方案
- 允许 Node.js 通过防火墙
- 或配置代理
```

### 查看完整日志

```bash
# 启动时保存日志
npm run dingtalk:multi 2>&1 | tee dingtalk.log

# 发送消息后，查看完整日志
cat dingtalk.log
```

- 发送测试消息
- 观察日志是否出现 `[Stream] ⭐ 收到机器人消息`

## 💡 测试技巧

### 最简单的测试方法

1. **先测单聊**
   - 添加机器人为好友
   - 发送"你好"
   - 看是否回复

2. **再测群聊**
   - 创建测试群
   - 添加机器人
   - @机器人发送消息

### 使用 HTTP 回调模式测试（备选）

如果 Stream 模式一直不行，可以尝试 HTTP 回调：

1. 使用 `dingtalk-webhook.js`
2. 配置 ngrok 或内网穿透
3. 在开放平台填写回调 URL
