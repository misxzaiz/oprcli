# 约束条件和重要规则

本文档包含所有必须遵守的重要约束和规则。

## 🚫 严格禁止修改

### 禁止目录

```
D:/space/base/oprcli/    → 整个基础版本目录（禁止修改）
```

**包括但不限于**：
- ❌ 任何 `.js` 文件
- ❌ 任何配置文件（`.env`, `.json` 等）
- ❌ 任何文档文件（`.md` 等）
- **特别禁止**：关闭或干扰运行在端口 **12840** 的应用

### 为什么禁止？

- **基础版本**：稳定版本，不应被修改
- **端口保护**：12840 端口的应用必须保持运行
- **系统稳定性**：避免意外破坏

## ✅ 可以修改

### 可修改目录

```
D:/space/oprcli/         → 当前工作版本的任何文件
```

**包括**：
- ✅ `server.js` - 主服务器
- ✅ `connectors/claude-connector.js` - Claude 连接器
- ✅ `connectors/iflow-connector.js` - IFlow 连接器
- ✅ `integrations/` - 所有集成模块
- ✅ `utils/` - 所有工具模块
- ✅ `system-prompts/` - 系统提示词
- ✅ `.env` - 环境配置
- ✅ `scheduler/` - 定时任务模块

## ⚠️ 安全意识

### 操作前确认

1. **确认文件路径**
   - 检查是否在 `D:/space/oprcli/` 下
   - 确认不在 `D:/space/base/oprcli/` 下

2. **敏感操作前告知用户**
   - 删除文件前
   - 修改核心配置前
   - 执行危险命令前

3. **端口检查**
   - 禁止关闭端口 12840 的应用
   - 禁止修改基础版本的配置

- **项目目录**：`D:/space/oprcli/`（可修改）
- **基础目录**：`D:/space/base/oprcli/`（禁止修改）

| **12840** | 基础版本（base） | 保护状态 | ❌ 禁止修改 |

## 📝 交互准则

### 钉钉适配

**为什么重要**：
- 用户通过钉钉机器人访问
- 移动端屏幕小，需要简洁
- Markdown 渲染支持有限

**最佳实践**：
- ✅ **简洁明了**：适合移动端快速阅读
- ✅ **结构清晰**：使用列表、代码块、分隔线
- ❌ **避免表格**：钉钉 Markdown 渲染支持有限
- ✅ **分段输出**：长内容分段发送，避免信息过载

### 主动分析

**原则**：
- 理解用户意图，主动提供解决方案
- 遇到问题时，先查阅帮助文档
- 提供多种方案，说明优缺点

### 工具使用

**优先级**：
1. Claude Code 原生工具
2. MCP Browser 工具
3. 自定义 Skills

**注意事项**：
- 遵循工作目录规范
- 不要硬编码敏感信息
- 使用环境变量配置

## 🔒 数据安全

### 敏感信息处理

**不要暴露**：
- ❌ Webhook URL
- ❌ API 密钥
- ❌ 密码和 Token
- ❌ 个人隐私信息

**正确做法**：
- ✅ 使用环境变量
- ✅ 使用配置文件
- ✅ 参考配置模板（`.env.example`）

### 配置文件

**git 提交前检查**：
- `.env` - 不要提交（包含敏感信息）
- `.env.example` - 可以提交（配置模板）
- `scheduler/tasks.json` - 不要提交（可能包含敏感配置）

## 🚨 紧急情况处理

### 如果误操作

**如果不小心修改了禁止目录**：
1. 立即停止操作
2. 告知用户发生了什么
3. 寻求帮助恢复

**如果端口冲突**：
1. 检查端口占用：`netstat -ano | findstr :12840`
2. 不要强制关闭
3. 联系用户处理

### 获取帮助

**文档查阅**：
```bash
# 查看文档索引
cat system-prompts/docs/README.md

# 查看故障排查
cat system-prompts/docs/troubleshooting.md
```

**用户命令**：
- `help` - 查看可用命令
- `status` - 查看系统状态
