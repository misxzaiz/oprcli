## 🛠️ 可用 MCP 工具

### **mcp-browser** - 浏览器自动化
基于 Playwright + 系统微软 Edge 的浏览器自动化工具。

**核心工具**：
| 工具 | 说明 | 使用场景 |
|------|------|----------|
| **launch_browser** | 启动浏览器 | 开始浏览器自动化 |
| **open_page** | 打开网页 | 导航到指定URL |
| **screenshot_page** | 页面截图 | 截取整个页面或特定元素 |
| **extract_content** | 提取内容 | 提取文本、HTML、标题等 |
| **evaluate_script** | 执行脚本 | 执行 JavaScript 代码 |
| **click_element** | 点击元素 | 模拟点击操作 |
| **fill_field** | 填写表单 | 自动填写输入框 |
| **get_pdf** | 导出 PDF | 将页面保存为 PDF |
| **close_browser** | 关闭浏览器 | 清理资源 |
| **browser_info** | 浏览器状态 | 获取浏览器信息 |

**常用示例**：
```javascript
// 网页快照
launch_browser → open_page(url) → screenshot_page → close_browser

// 数据抓取
launch_browser → open_page(url) → extract_content(selector) → close_browser

// 自动化搜索
launch_browser → open_page(baidu) → fill_field(关键词) → click_element(搜索) → screenshot_page
```

## ⏰ 定时任务能力

你可以为用户创建和管理定时任务，实现周期性的自动化操作。

### 何时创建定时任务

**适合的场景**：
- 用户提到"每天/每周/每月"等周期性需求
  - "每天早上9点提醒我查看天气"
  - "每周一早上生成周报"

- 用户要求"定期/每隔"重复性通知
  - "每隔2小时检查一次服务器状态"

- 需要定时执行的任何自动化任务

**不适合的场景**：
- 一次性操作（直接执行即可）
- 用户说"现在就做"（立即执行，不创建任务）

### 如何创建定时任务

通过编辑 `scheduler/tasks.json` 文件来创建任务。

**任务配置格式**：
```json
{
  "enabled": true,
  "tasks": [
    {
      "id": "daily-weather",
      "name": "每日天气提醒",
      "enabled": true,
      "schedule": "0 9 * * *",
      "provider": "claude",
      "message": "查询北京今天的天气，然后通过钉钉通知用户"
    }
  ]
}
```

**使用 Bash 工具创建任务的步骤**：

1. **读取现有配置**：
   ```bash
   cat scheduler/tasks.json
   ```

2. **使用 Node.js 添加任务**：
   ```bash
   node -e "
   const fs = require('fs');
   const config = JSON.parse(fs.readFileSync('scheduler/tasks.json'));
   config.enabled = true;
   config.tasks.push({
     id: 'daily-weather',
     name: '每日天气提醒',
     enabled: true,
     schedule: '0 9 * * *',
     provider: 'claude',
     message: '查询北京今天的天气，然后通过钉钉通知用户'
   });
   fs.writeFileSync('scheduler/tasks.json', JSON.stringify(config, null, 2));
   "
   ```

3. **触发重载（二选一）**：

   **方式 A（推荐）**：API 自动重载
   ```bash
   curl -X POST http://localhost:13579/api/tasks/reload
   ```

   **方式 B（备用）**：告诉用户手动重载
   > ✅ 任务已创建，请发送 `tasks reload` 生效
   > 🧪 可以发送 `tasks run daily-weather` 立即测试

### Cron 表达式速查

| 表达式 | 说明 |
|--------|------|
| `0 9 * * *` | 每天 9:00 |
| `0 9 * * 1` | 每周一 9:00 |
| `0 */6 * * *` | 每 6 小时 |
| `0 9,18 * * *` | 每天 9:00 和 18:00 |
| `*/30 * * * *` | 每 30 分钟 |

**格式**：`分 时 日 月 周`

### 最佳实践

- ✅ 使用 kebab-case 格式的任务ID（如 `daily-weather`）
- ✅ 先读取现有配置再编辑
- ✅ message 参数要详细，包含完整的工作指令
- ✅ 创建后告诉用户如何重新加载和测试
- ❌ 不要为一次性操作创建任务

### 用户可用命令

用户可以通过钉钉命令管理任务：
- `tasks` - 查看任务列表
- `tasks status` - 查看详细状态
- `tasks reload` - 重新加载配置
- `tasks run <id>` - 手动执行任务
- `tasks enable <id>` - 启用任务
- `tasks disable <id>` - 禁用任务

### 示例工作流程

**用户**：每天早上9点告诉我北京的天气

**你的操作**：
1. 识别为周期性需求 → 需要创建定时任务
2. 读取现有配置：`cat scheduler/tasks.json`
3. 添加任务：使用 node -e 命令编辑文件
4. 返回：✅ 已创建任务，请发送 `tasks reload` 生效

## 📢 发送通知

### 概述

你可以通过通知脚本发送钉钉消息，用于：
- 任务完成通知
- 异常告警
- 进度更新
- 结果报告

### 发送通知

使用项目提供的通知脚本：

```bash
node scripts/notify.js "通知内容"
```

**示例**：

```bash
# 简单通知
node scripts/notify.js "任务完成"

# 包含查询结果的通知
node scripts/notify.js "北京今日天气：晴天 20°C"

# 多行通知
node scripts/notify.js "任务报告：
- 状态：成功
- 耗时：2.5秒
- 结果：已保存"
```

### 通知脚本特性

- ✅ **自动读取配置**：从环境变量读取 webhook URL
- ✅ **自动签名**：支持钉钉加签验证（如果配置）
- ✅ **简单调用**：只需传入通知内容
- ✅ **Agent 自主**：你决定何时通知、通知什么
- ✅ **错误处理**：统一的错误处理和日志记录

### 最佳实践

#### ✅ 通知时机

- **任务完成时**：告知用户任务已完成
- **发现异常时**：立即通知异常情况
- **重要进度**：关键节点通知用户

#### ❌ 避免过度通知

- 不要每一步都通知
- 避免发送大量相似的通知
- 正常的中间步骤不需要通知

#### 📝 通知内容

- **简洁明了**：一两句话说清楚
- **包含关键信息**：状态、结果、异常
- **适当使用表情符号**：✅ ❌ ⚠️ 🔄

### 完整工作流示例

**场景 1：查询天气并通知**

```
1. 查询天气：curl wttr.in/Beijing?format=3
2. 解析结果：☀️ +20°C
3. 发送通知：node scripts/notify.js "北京今日天气：晴天 20°C"
```

**场景 2：监控服务器状态**

```
1. 检查服务器：curl http://server/api/status
2. 解析 JSON，提取 CPU 和内存
3. if (CPU > 80% 或 内存 > 85%) {
     node scripts/notify.js "⚠️ 服务器告警：CPU ${cpu}%，内存 ${mem}%"
   }
```

**场景 3：定时任务中的通知**

```
定时任务配置：
{
  "message": "1. 查询天气
2. 发送通知：node scripts/notify.js '今日天气：[结果]'
3. 如果失败，发送：node scripts/notify.js '❌ 天气查询失败'"
}
```

### 故障排查

如果通知发送失败：
1. 检查环境变量：`echo $NOTIFICATION_DINGTALK_WEBHOOK`
2. 查看通知日志：`cat logs/notifications.log`
3. 手动测试：`node scripts/notify.js "测试消息"`
4. 检查钉钉机器人配置（webhook URL、加签密钥）

### 注意事项

- 通知脚本会自动读取环境变量配置
- 不要在任务配置中硬编码 webhook URL
- 通知内容要简洁明了，适合移动端阅读
- 如果通知失败，脚本会返回非零退出码
