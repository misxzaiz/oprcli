## 📝 使用示例

### 添加新平台

现在添加新平台非常简单，只需5步：

```javascript
// 1. 创建新类继承基类
const BasePlatformIntegration = require('./base-platform-integration')

class WeChatIntegration extends BasePlatformIntegration {
  // 2. 实现连接方法
  async connect(messageHandler) {
    // 连接到微信平台
    this.client.on('message', async (msg) => {
      await messageHandler(msg)
    })
  }

  // 3. 实现发送方法
  async send(target, message, originalMessage, type) {
    return this.sendWithRetry(target, message, async (tgt, msg) => {
      // 实际发送逻辑
    })
  }

  // 4. 实现消息解析方法
  extractContent(rawMessage) {
    return rawMessage.content
  }

  getConversationId(rawMessage, type) {
    return rawMessage.chatId
  }

  getReplyTarget(rawMessage) {
    return rawMessage
  }

  getMessageId(rawMessage, type) {
    return rawMessage.id
  }
}

// 5. 在 server.js 中使用
const wechat = new WeChatIntegration(config, logger, rateLimiter)
await wechat.init(handleWeChatMessage)
```

## ⚠️ 暂时移除的功能

为保持最小可用集，以下功能已暂时移除：

| 功能 | 原因 | 影响 |
|------|------|------|
| 自动清理定时器 | 非核心 | 内存会缓慢增长 |
| send_file处理 | QQ特有 | QQ无法发送文件 |
| 会话持久化 | 可选 | 服务重启会话丢失 |
| BoundedMap | 简化 | processedMessages可能无限增长 |

### 何时添加回这些功能？

- ✅ **优先级高**: 添加自动清理（防止内存泄漏）
- ✅ **优先级中**: 恢复会话持久化（提升用户体验）
- ⏸️ **优先级低**: 恢复send_file（仅在需要时）

### 3. 添加自动清理（如果内存增长）

```javascript
// 在 BasePlatformIntegration 中添加
startAutoCleanup() {
  this.cleanupTimer = setInterval(() => {
    this.performCleanup()
  }, 5 * 60 * 1000)  // 5分钟
}

performCleanup() {
  const now = Date.now()
  const oneHour = 60 * 60 * 1000
  const oneDay = 24 * 60 * 60 * 1000

  // 清理1小时前的消息
  for (const [msgId, time] of this.processedMessages) {
    if (now - time > oneHour) {
      this.processedMessages.delete(msgId)
    }
  }

  // 清理24小时前的会话
  for (const [convId, session] of this.conversations) {
    if (now - session.startTime > oneDay) {
      this.conversations.delete(convId)
    }
  }
}
```
