# OPRCLI 永久记忆系统

## 🎯 项目概述

为OPRCLI（统一AI CLI连接器）实现的永久记忆系统，支持自动记录、智能总结和快速搜索。

## ✨ 核心特性

- ✅ **自动记录**：自动记录所有对话（用户消息 + AI回复）
- ✅ **分层总结**：5层时间金字塔，从详细到概要
- ✅ **智能搜索**：支持关键词、标签、时间范围搜索
- ✅ **Load命令**：一键触发记忆总结和整合
- ✅ **轻量级**：JSON存储，无需数据库
- ✅ **易集成**：最小化改动OPRCLI核心代码

## 📊 系统架构

```
┌─────────────────────────────────────┐
│      OPRCLI 对话系统                 │
└──────────────┬──────────────────────┘
               │ 自动记录
               ▼
┌─────────────────────────────────────┐
│    原始记忆层 (Raw Memories)          │
│  - 永久保存所有对话                   │
│  - 支持搜索和标签                     │
└──────────────┬──────────────────────┘
               │ Load 命令
               ▼
┌─────────────────────────────────────┐
│    总结记忆层 (Summaries)             │
│  Level 0: 1小时范围（详细）           │
│  Level 1: 1天范围（较详细）           │
│  Level 2: 1周范围（中等）             │
│  Level 3: 1月范围（简略）             │
│  Level 4: 1季度范围（概要）           │
└─────────────────────────────────────┘
```

## 🚀 快速开始

### 1. 测试安装

```bash
cd D:/temp/oprcli
node test-memory.js
```

### 2. 启动OPRCLI

```bash
npm start
```

### 3. 使用命令

在钉钉中发送：

```
load              # 触发记忆总结
search 钉钉       # 搜索记忆
stats             # 查看统计
tag abc123 重要   # 添加标签
```

## 🔧 核心功能

### 1. 自动记录

系统自动记录所有对话：
- 用户消息 → 标签：`user`, `message`
- AI回复 → 标签：`assistant`, `response`

### 2. Load 命令

一键触发记忆总结：
```
load
```

**功能**：
1. 总结未总结的记忆
2. 整合已总结的记忆
3. 生成分层结构

### 3. 搜索功能

```
search 钉钉
搜索 机器人
```

支持：
- 关键词搜索
- 标签过滤
- 时间范围
- 结果限制

### 4. 统计信息

```
stats
统计
```

显示：
- 记忆总数
- 总结层级分布
- 存储占用

## 📊 性能

### 存储效率
- 原始记忆：~200 bytes/条
- Level 0 总结：~38 bytes/条
- Level 1 总结：~16 bytes/条
- 压缩比：5-10x

### 时间复杂度
- 创建记忆：O(1)
- 搜索记忆：O(n)
- Load 命令：O(n log n)

## 🎯 使用场景

### 1. 项目记录
自动记录项目讨论、决策、问题解决过程

### 2. 知识积累
长期积累AI交互经验，形成知识库

### 3. 快速检索
需要时快速查找历史对话和解决方案

### 4. 自动总结
定期总结，避免信息过载

## 💡 最佳实践

1. **定期 Load**：每天1-2次，避免记忆堆积
2. **标签管理**：重要对话手动添加标签
3. **定期备份**：备份 `data/memory/` 目录
4. **监控统计**：定期查看 `stats` 了解系统状态

## 🔄 数据流程

```
用户对话
  ↓
自动记录 → 原始记忆层
  ↓
Load 命令
  ↓
分组总结 → Level 0 (1小时)
  ↓
合并升级 → Level 1-4
  ↓
压缩存储 → 节省空间
```
